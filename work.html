dashboard-bb: Flexible Dashboard for Network Management

Application prototype for Jumpin

Homepage: dashboard-bb.herokuapp.com

Github: dashboard and dashboard-api

dashboard-bb is an application prototype implemented for the crowdsourcing project platform Jumpin. It is a dashboard for managing dynamic widgets fed with network data from various input feeds. It is flexible in the extent that it is possible to easily add new widgets, change the data API and visually change the setup as desirable.

A big part of the application visual design was conceived by PT Inovação, including a Photoshop prototype. I implemented it from scratch using own tools and technologies. It started out as a testbed to learn new modern web languages/technologies such as CoffeeScript, Backbone.js, Yeoman and later MongoDB. The context of dynamic widgets in a frontend app seemed the perfect application for this sort of technology.

Nevertheless, this project stretches over the whole stack, frontend to backend. The frontend is a single page web application built on top of Backbone.js and Gridster.js. It is structured so that it is independent from the data it pulls from a remote RESTful JSON API, and each widget has its own data and behaviour. In order to optimize coding performance and runtime performance, I used the (terrific) Yeoman tool along with the styling magic of Sass, Compass and Bootstrap.

The backend and API, while not included in the specification, was simply developed in order to have a functional prototype and simulate a real setup and data. It is built with Ruby on Rails and MongoDB as an example of a perfomant and easy to scale setup.

Front-end

Application Structure

Backbone.js + Underscore.js

  • Model the collection of widgets, and each individually
  • Separate control and event handling of each building block: refresh widget once its model is updated
  • Rich set of enumerable functions to filter/extract data from collections - courtesy of Underscore.js
  • Widget's position, size and data, persisted into the client's local storage with backbone.localStorage
  • Categories and widget data fetched from the RESTful JSON API
  • Serialize form data into Backbone models with bakcbone.Syphon
  • Url routing with pushState

CoffeeScript + Eco templates

  • Backbone + CoffeeScript = just feels right and fun
  • Object orientation keeps widget's base functionality in super class while the specific functionality in sub classes
  • Eco templates render rich views with the cool syntax of CS - Ruby's ERB cousin

Semantic Layout

HTML5 + HTML5 Boilerplate

  • Fullscreen and Local Storage APIs
  • Better semantics and form inputs
  • Boilerplate template with normalizations and helpers for an adaptable web site

Twitter Bootstrap

  • Well designed, tried-and-true components: navbar, modal screen, popover, tooltip, tables, tabs, responsive grids and images,...
  • Less time designing, more time styling
  • Mobile first, responsive design, cross-platform
  • Flexible, fully customizable and well documented
  • Sass

Styling

CSS3 + Sass + Compass

  • Gradients, columns, border radiuses, transitions and transforms: card flip effect, custom scroll bars
  • Nesting of selectors = "syntactically awesome"!
  • Organize stylesheets with minimal HTTP requests
  • Define and reuse variables
  • Pre-built mixins and helpers

FontAwesome

  • Minimize use of images
  • Reasonable collection of scalable vector icons
  • Customizable: color, size, rotations and combinations

Added Functionality

Gridster.js

  • Wonderfully designed drag & drop block grid that fits perfectly with a widget dashboard
  • Fully customizable: move handler on widget's front face; resize handler on the back
  • Event callbacks for resize and reposition used to save dashboard setup
  • jQuery

jQuery

  • Used all over the app for DOM manipulation and simple animations
  • Seamless integration with Backbone

Graph.js

  • Minimal, well designed HTML5 graphs with animations
  • Super simple to setup
  • Graph canvas resized on widget resize

html2canvas

  • Render DOM elements into Base64 images - used to take screenshots of the current dashboard setup
  • Gradient CSS not supported - yet

Back-end

API

Rails API

  • Subset of a regular Ruby on Rails 4: no middleware and view templates
  • Simply serve data as JSON
  • Action Mailer for sending HTML emails with screenshot attached

MongoDB + mongoid

  • NoSQL database for simple and easy modeling of objects
  • Direct repesentation of JSON objects served by the API
  • Highly scalable
  • Mongoid provides Rails-like relations and dynamic queries that are easy to read and understand

PTSense: improving Quality of Experience in Public Transportation

Master thesis project @ FEUP & Imperial College London

Homepage: PTSense

Github: PTSense

PTSense is a proof of concept Android application, part of an academic research. It is the front-end of a crowdsourcing sensing framework that collects affective and context data from commuters across the public service network.

By making inferences over the sensed data and feedback it is possible to create customer profiles and match them against real-time participatory sensing to serve customers better. For instance, routing users to more attractive routes before they need it (eg. less crowded; not necessarily faster). This real-time sensing across the network brings not only benefits to the individual but for everyone using the system.

This project is fully documented in my Thesis page, with a brief summary here.

Edit: Moovit is a new public transit mobile app that already follows this crowdsourcing concept. It allows the share of public transit conditions by the users of the system, in real-time. Nevertheless, our research framework and mobile app would do this in a less pervasive way, using sensors, wearable computing and a stronger machine-learning module in the cloud that could understand and transform the vast sensor data into quality of experience. Even more, predicting the use of public transportation and making personalized route suggestions.

Mobile Application

Platform

Android + Java + SQlite

  • Use of smartphone sensors and Android API to access them
  • Interact with notifications as the gateway to the app
  • Sensing in a background service, refreshing the UI if visible
  • Store raw data in device's database before sending to server
  • Send data as JSON - sensor data and trip feedback
  • Customizable settings: enable/disable sensors, connect devices, set public tranportation service, and more
  • Simple UI: drag & drop sliders
  • Backward compatiblity from Android 2.1 to 4.0+ (ICS)
